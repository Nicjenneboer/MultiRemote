Detailed Description

a direct-effect port_in_out

This is a version of port_in_out where the various operations have direct (immediate) effect: no refresh or flush calls needed. This have a negative effect on the execution time.

The preferred way to use this class is through the overloaded constructor function direct().

flush the port buffer

This function flushes the port write buffer: if a buffered value has not yet been written it is written to the pin. When the value has been written, it might be written again.

The pin write buffer might also be flushed as a side-effect of other operations.
