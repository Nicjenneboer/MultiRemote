hwlib
C++ OO-style close-to-the-hardware library
Graphics

The (abstract) types hwlib::istream istream and hwlib::ostream ostream are used to read and write characters.


xy

An xy is a value type that stores two int__fst16_t values x and y. xy values can be added and subtracted, and can be multiplied and divided by an integer value. An xy can also be printed to an ostream using operator<<.

attributes and operations meaning or effect
x x value
y y value
+ adds two xy values
- subtracts to xy values
* multiplies the x and y value by an integer
/ divides the x and y values by an integer
<< prints an xy to an ostream

The overloaded all() function can be used to iterate over all (x,y) values within the (0 .. x-1, 0 .. y-1) range (in some unspecified order).


color

A color is a value type that stores a color as three 8-bit red, green and blue values, plus a 'is_transparent' flag. When the transparent flag is not set, the color values determine the color. When it is set, the color values have no meaning. An color can also be printed to an ostream using operator<<.

Colors can be constructed, negated, and compared for equality or inequality.

attributes and operations meaning or effect
is_transparent transparency flag
rad red intensity
green green intensity
blue blue intensity
color(r,g,b) construct a color from its components
color(v) construct a color from its 24-bit RGB value
- yields the inverse of a color
== tests for equality
!= tests for inequality
<< prints an xy to an ostream

The following color constants are available:

  • black
  • white
  • red
  • green
  • blue
  • gray
  • yellow
  • cyan
  • magenta
  • transparent
  • violet
  • sienna
  • purple
  • pink
  • silver
  • brown
  • salmon


image

An hwlib::image image is an abstract class that defines an interface to a picture, that is: a rectangle of read-only pixels. An image is used to embed a picture in the application.

attributes and operations meaning or effect
size size in pixels in x and y direction
hwlib::image::operator[] operator [] the color of the pixel at location loc


font

An hwlib::font font is an abstract class that defines an interface to a set of pictures that show characters as a graphic images. A font is used to implement a character terminal on a graphic hwlib::window window.

attributes and operations meaning or effect
operator[ c ] returns the image for char c

Two concrete built-in font classes are available:

  • hwlib::font_default_8x8 font_default_8x8
  • hwlib::font_default_16x16 font_default_16x16

The font_default_16x16 is not available on AVR8 targets because the AVR compiler can't handle it.


window

An hwlib::window window is an abstract class that defines an interface to a graphics display. The display can be cleared, and a pixel can be set to a color.

Window operations are (potentially) buffered: a subsequent flush() call is required in order for the previous operations to take effect.

attributes and operations meaning or effect
size size of the display, in pixels in a x any direction
background the background color
foreground the foreground color
clear() write the background color to all pixels
clear(col) write color col to tall pixels
write(loc, col) write color col to the pixel at loc
write(loc, img) write image img to the location loc
flush() flush all pending changes to the window

The window_part decorator creates a window in a rectangular part of another window.

The window_invert decorator creates a window that writes inverted (the color of each pixel negated) to the underlying window.


drawables

A hwlib::drawable drawable is an abstract class that defines an interface for something that cab be drawn on a hwlib::window window.

attributes and operations meaning or effect
start origin (top left corner) of where the drawable is to be drawn
draw(w) draw the drawable on window w

A line is a drawable. It is created by specifying its origin and its endpoint. A color can be specified. If none is, the foreground color of the window is used.

A circle is a drawable. It is created by specifying its midpoint and its radius A color can be specified. If none is, the foreground color of the window is used.


terminal_from

A terminal_from creates a character terminal from a window and a character font.
